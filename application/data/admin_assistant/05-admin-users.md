# Administrators, roles, API timing / 管理员、权限与定时任务

## 1) 创建后台管理员

1. 打开 **系统 → 管理员**（或 **管理员列表**，名称因皮肤略有差异）。  
2. 点击 **添加**。  
3. 填写 **用户名、密码**（强密码），选择 **角色**。  
4. 保存。将登录地址与账号交给同事时，勿通过公开渠道发送密码。

## 2) 角色与权限

1. 打开 **系统 → 角色 / 用户组**（常见名：**角色管理**）。  
2. **新增角色**，在权限树中勾选可访问的 **菜单/操作**（如仅内容、仅采集、仅模板等）。  
3. 将角色分配给管理员。  
4. **超级管理员**（常见为 `admin_id = 1`）通常 **不受权限树限制** —— 该账号务必 **双人保管、强密码、勿共享**。

若某账号提示 **无权限**：由超级管理员在 **角色权限** 中补勾对应控制器/菜单。

## 3) 定时任务（采集、清理、外链等）

- MacCMS 常通过 **`api.php`** 暴露 **定时任务** 接口，由宝塔/crontab **定时 HTTP 访问**。  
- **安全**：生产环境应为 `api.php` **限制 IP**、使用 **密钥参数**（若版本支持）、避免公网扫描。  
- 任务未执行：检查服务器 crontab 是否报错、HTTPS 证书、防火墙、以及 CMS 内 **定时任务开关/绑定 URL**。

## 4) 日志与审计

- **管理员操作日志 / 入库日志**：用于追溯谁改了配置或批量删除了数据（具体菜单以版本为准）。  
- 定期备份日志表或归档，以免表过大。

## English summary

Admins live under **System → Administrators**; permissions via **Roles**. Protect **admin_id = 1**. Lock down **`api.php` timing** URLs (IP allowlist, secrets). Review logs periodically.
