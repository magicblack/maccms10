# Sidebar modules: 基础 / 内容 / 用户 / 模版 / 采集 / 数据库 / 应用

以下名称来自语言包 **menu/*** 键，界面应以当前皮肤为准。

---

## 基础 (`menu/base`)

| 菜单 | 用途 |
|------|------|
| **分类管理** `type` | 全站 **类型与分类**（视频/文章/专题等栏目树），内容发布前 **必须先有分类**。 |
| **专题管理** `topic` | 专题 CRUD、关联视频/文章等。 |
| **友链管理** `link` | 友情链接。 |
| **留言管理** `gbook` | 留言本审核与删除。 |
| **评论管理** `comment` | 全站评论审核。 |
| **附件管理** `images` | 已上传图片/附件浏览与维护。 |

---

## 视频 (`menu/vod`)

| 菜单 | 用途 |
|------|------|
| **服务器组** `server` | 多线路/多源播放时的 **server 标识** 与解析入口配合。 |
| **播放器** `player` | **解析代码**、排序、编码；与 **系统 → 播放器参数**、视频表单中 **播放组** 对应。 |
| **下载器** `downer` | 下载方式或下载插件配置（依版本）。 |
| **视频数据** `vod_data` | 主列表：筛选、编辑、删除、推荐、锁定。 |
| **添加视频** `vod_add` | 单条发布。 |
| **无地址视频** `vod_data_url_empty` | 排查缺播放地址的条目。 |
| **已锁定 / 未审核 / 需积分 / 有分集剧情** | 条件子集列表，便于运营处理。 |
| **批量操作视频** `vod_batch` | 批量改分类、状态等。 |
| **重名视频数据** `vod_repeat` | 去重与合并参考。 |

**剧情连载**：部分版本有 **剧情** 模块（URL 配置里可见 plot）；**有分集剧情** 列表用于维护分集与正片关联。

---

## 文章 (`menu/art`)

- **文章数据**、**添加文章**、锁定/未审核/批量/重名 — 与视频逻辑类似，模块换为 **文章**。

---

## 演员库 / 角色库 (`actor` / `role`)

- 人物、角色资料维护；可与 **视频关联演员**、前台 **明星/角色** 页展示（视模板）。

---

## 网址 (`website`)

- **网址数据**、**添加网址**、锁定/未审核/批量/重名 — 导航站、资源站收录类内容。

---

## 漫画 (`manga`)

- 若安装/开启漫画模块：分类、列表、添加、**无地址漫画**、**需积分漫画** 等（菜单项可能随版本增减）。  
- 章节、阅读地址格式以 **添加/编辑表单说明** 为准（常见多话数分隔规则）。

---

## 用户 (`menu/users`)

| 菜单 | 用途 |
|------|------|
| **管理员** `admin` | 后台账号。 |
| **会员组** `group` | 权限、积分、收费策略分组。 |
| **会员** `user` | 前台用户列表、封禁、调积分。 |
| **充值卡** `card` | 卡密生成与核销。 |
| **会员订单** `order` | 支付订单查询。 |
| **访问日志** `ulog` | 用户访问/播放等行为日志（依版本字段）。 |
| **积分日志** `plog` | 积分变动。 |
| **提现记录** `cash` | 会员提现审核。 |

---

## 模版 (`templates`)

| 菜单 | 用途 |
|------|------|
| **模板管理** `template` | 选择/编辑 PC、WAP 模板文件。 |
| **广告位管理** `ads` | 广告位与模板内调用标识。 |
| **标签向导** `wizard` | 生成常用 **标签代码** 片段。 |

---

## 生成 (`make`)

- **生成选项** `make_opt`：路径、并发等。  
- **生成首页 / WAP 首页 / 地图** `make_index` 等：静态化或预渲染（以程序实现为准）。  
- 使用 **静态/缓存** 后，改模板或大事数据后需 **重新生成**。

---

## 采集 (`cjs`)

| 菜单 | 用途 |
|------|------|
| **推荐资源** `union` | 推荐合作资源站或列表（依版本）。 |
| **定时挂机** `collect_timming` | 采集计划与挂机说明。 |
| **自定义接口** `collect` | API 型采集节点。 |
| **自定义规则** `cj` | 规则型采集。 |

---

## 数据库 (`db`)

- **数据库管理**：备份、恢复、优化（以界面为准）。  
- **执行 SQL**：高危，仅维护人员且 **先备份**。  
- **数据批量替换**：批量改字段内容，易误伤，需 **where 条件** 与备份。  
- **挂马检测**：扫描可疑脚本/SQL（辅助工具，非 100%）。

---

## 应用 (`apps`)

- **应用市场** `addon`：安装/升级/禁用插件；插件可能新增 **菜单** 与 **定时任务**，需在 **角色权限** 中授权。

---

## English summary

**Base** = taxonomy + topic + link + gbook + comment + annex. **VOD** = server + player + downer + many filtered lists. **Users** = admin + group + end-users + card/order/logs/cash. **Templates** = files + ads + wizard. **Collect** = union + timing + custom API + rules. **DB** = backup + SQL + replace + scan.
