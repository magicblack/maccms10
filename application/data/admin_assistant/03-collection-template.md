# Collection, templates, cache / 采集、模板与缓存

## A. 采集（对接资源站）

### 1) 采集前准备（必做）

1. 打开 **系统 → 采集参数配置**。  
2. 检查各 Tab（视频/文章/…）中的 **数据状态**（如：未审核 / 已审核）、**同步图片**、**随机点击/评分** 等是否符合运营要求。  
3. 保存。大批量采集后再改规则，往往需 **重新处理已入库数据**，成本高。

### 2) 添加采集节点（常见流程）

1. 进入 **采集** 菜单下的 **自定义接口** 或 **自定义规则**（名称因版本略有不同）。  
2. **新增** 节点：填写资源站提供的 **接口地址**、编码、请求参数等。  
3. **绑定分类映射**：将资源站的类型ID映射到 **本地类型/分类**；映射错误会导致内容进错栏目或无法入库。  
4. 先 **测试** 单页或少量数据，确认封面、播放地址、备注字段正常。  
5. 再执行 **当天/全量/断点续采**（以界面按钮为准）。

### 3) 常见问题

- **无播放地址 / 解析失败**：检查播放器配置（**系统 → 播放器**）与资源站格式是否匹配。  
- **图片不显示**：检查 **附件/远程图片** 设置与 **采集参数** 中是否开启同步图。  
- **重复数据**：使用 **重复数据** 或 **批量操作** 清理；规则上可配合唯一指标（如来源ID）。  

### 4) 萌芽采集 / Teek 等插件

若使用第三方采集插件，安装与卸载以 **插件官方文档** 为准；核心 CMS 的 **采集参数配置** 仍可能影响入库默认值。

---

## B. 模板（前台外观）

1. **系统 → 网站参数** 或 **模板** 菜单中确认 **PC / WAP 模板目录**。  
2. 模板文件位于站点根目录 **`template/{模板名}/`** 下（含 html、css、js）。  
3. 修改后：  
   - 若开启 **页面缓存 / 静态生成**，需到 **生成** 或 **缓存** 相关菜单 **更新缓存** 或 **重生成**。  
   - 浏览器 **强制刷新 (Ctrl+F5)** 避免旧 JS/CSS。  
4. 标签语法以 MacCMS 官方标签文档为准（如 `{$maccms.site_name}`、`{maccms:site}` 等，具体以您模板实际版本为准）。

---

## C. 静态与缓存

- **全站缓存 / 页面缓存**：提高性能，但会导致 **改模板或配置后短期内仍看旧页** → 需清理。  
- **浏览器 CDN**：若有 Cloudflare 等，改模板后可能还需刷新 CDN。  

---

## English summary

Before bulk **collect/import**, set **System → Collect parameters**. Map remote categories correctly. Use **Templates** under `template/`; clear **cache/static** after changes. Test one node before full sync.
